byte Configuration::lineFollowCorrectionMinus

Defines the value that shall be subtraced from the current speed in case of a correction.

In case of a correction one motor needs to rotate slower than the other one. Depending if a correction to the left or to the right side is needed, this positive value will be added to the left or right motor.
If the car drives away from the line left hand side the right line sensor will measure white ground. The starting correction will now subtract the value defined in lineFollowCorrectionMinus from the current speed of the right motor. If the car drives away from the line to the right hand side the value will be added to the left motor. Important: The positive value defined here will be subtracted.

Definition at line 64 of file Configuration.h.

byte Configuration::lineFollowCorrectionPlus

Defines the value that shall be added to the current speed in case of a correction.

In case of a correction one motor needs to rotate faster than the other one. Depending if a correction to the left or to the right side is needed, this positive value will be added to the left or right motor.
If the car drives away from the line left hand side the right line sensor will measure white ground. The starting correction will now add the value defined in lineFollowCorrectionPlus to the current speed of the left motor. If the car drives away from the line to the right hand side the value will be added to the right motor.

int Configuration::lineFollowWhiteThreshold

Threshold value from black to white.

If the sensor measures a value below that threshold value than the software considers that sensors physical position to be on "white" ground. This treshold value is calculated during the calibration of the sensors.
The default value 156 is based on first measurements and set as long as no calibration is done.
