void CubeApproach::doJob ( boolean  followLine)

Does the job for which this class was written.
This method has to be called in regular intervals.
The sensor data of the distance sensors is captured and based on the result the car is controlled to either follow the line or drive to the cube.

Important: Call startIt() first!

int CubeApproach::_stopCarExtMoveCommandId [private]

The id of the ext move command that is responsible for stopping the car after the cube has been detected the first time.

Per default this value is set to -1. This means, that the command has not yet been executed. The doJob() method will therefore execute this command. After starting it's execution doJob() will wait until the execution of this command has finished. Then the second part of the cube approach algorithm (second part: cube approach without following the line) starts.
